# 08-new-pipeline

This change defines the project’s new slide pipeline: `source slide -> AI orchestrator -> strict JSON schema -> dumb HTML renderer -> GSAP -> HyperFrames -> QA Diff Loop`.

## Why This Change Exists

The previous workflow attempted to use HTML/CSS directly as the source of truth for slide layout. This proved to be extremely brittle and practically impossible to algorithmically diff or iterate upon. We needed a single, explicit pipeline contract that completely decouples the **layout geometry** from the **rendering engine**.

## What This Change Introduces

- A strict JSON layout schema (`scene_layout.json`) as the absolute source of truth.
- An AI Orchestrator (`orchestrator.py`) that uses the Anthropic Vision API to perform layout extraction automatically from high-res source images (downsampled via PIL).
- A "dumb" renderer script (`render_html_from_layout.py`) that blindly translates JSON coordinates into absolute CSS positioning.
- A deterministic GSAP animation layer mapped on top of the generated DOM.
- A QA Diff Loop that iterates the JSON layout until the pixel diff score drops below a specific threshold (e.g., `< 18.0`).

## Pipeline Summary

1. **Visual Analysis & Semantic Extraction**
   - Read the input slide image using `orchestrator.py` (Anthropic Vision API)
   - Downsample massive images using `PIL` to prevent API connection crashes
   - Extract `scene_layout.json` as the strict absolute layout schema (id, type, x, y, w, h, zIndex, style)

2. **Scene Rebuild (Dumb Renderer)**
   - Recreate the slide using the `render_html_from_layout.py` dumb renderer
   - Produce `custom-html.html`, mapping the JSON absolute coordinates directly to CSS

3. **Animation**
   - Build a deterministic GSAP timeline from storyboard data
   - Keep motion sequence-based and render-safe

4. **Render**
   - Use HyperFrames to export MP4 output
   - Preserve the `scene_layout.json` as the source of truth

5. **QA & Iteration Loop**
   - Compare the source and rendered thumbnails
   - Produce `qa/qa_report.md`
   - Iterate the JSON schema until the layout diff score is `< 18.0`.

## Required Artifacts

- `user/assets/slides/slide-N/scene_layout.json`
- `user/assets/slides/slide-N/custom-html.html`
- `scene/index.html`
- `scene/style.css`
- `scene/animation.js`
- `qa/qa_report.md`

## Skill Pack

The change is backed by the following local skills:

- `slide-scene-rebuild-html-skill` (Enforces the strict JSON source of truth rule)
- `gsap-storyboard-animator`
- `hyperframes-video-renderer`
- `visual-qa-checker`
- `project-orchestrator`

## Implementation Notes

- HTML MUST NOT contain manual layout logic; it should only ever be generated by the `render_html_from_layout.py` script.
- Keep outputs workspace-relative and predictable across macOS, Linux, and Windows.
- Keep the animation timeline deterministic so render output is stable.
