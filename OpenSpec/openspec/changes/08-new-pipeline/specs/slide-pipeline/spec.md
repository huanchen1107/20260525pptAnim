## ADDED Requirements

### Requirement: End-to-end slide pipeline
The system SHALL provide a repeatable slide pipeline that accepts slide source input and produces a sequence of inspectable artifacts for visual analysis, strict JSON schema layout extraction, dumb HTML rendering, GSAP animation, HyperFrames export, and QA diffing.

#### Scenario: Pipeline produces staged artifacts
- **WHEN** a user runs the orchestrator pipeline on a source slide image
- **THEN** the pipeline SHALL produce separate artifacts: `scene_layout.json`, `custom-html.html`, scene scripts, MP4 render output, and QA thumbnails
- **AND** each artifact SHALL be usable as input to the next stage without requiring the user to reconstruct missing context

#### Scenario: Users can inspect intermediate stages
- **WHEN** a stage completes successfully
- **THEN** the stage’s artifact SHALL remain available for inspection and rerun
- **AND** the pipeline SHALL not require the user to regenerate earlier stages just to review the output of a later stage

### Requirement: JSON Schema-first layout geometry
The system SHALL treat a strict JSON schema (`scene_layout.json`) as the absolute source of truth for all slide geometry, removing layout responsibility from HTML/CSS.

#### Scenario: Dumb HTML rendering
- **WHEN** the pipeline rebuilds a slide into scene files
- **THEN** the `custom-html.html` file SHALL be blindly generated by translating the absolute `x`, `y`, `w`, `h` properties from `scene_layout.json` into CSS
- **AND** the HTML MUST NOT make arbitrary layout assumptions or adjustments

#### Scenario: Storyboard guides animation
- **WHEN** storyboard data is available
- **THEN** the animation stage SHALL use that storyboard to sequence the scene via a deterministic GSAP timeline
- **AND** the resulting output SHALL be suitable for deterministic HyperFrames video rendering

### Requirement: QA Diff Loop Threshold
The system SHALL iteratively test the rendered output against the source slide thumbnail until it meets an objective visual quality threshold.

#### Scenario: Iterative pixel-diff QA
- **WHEN** the pipeline renders a final preview thumbnail
- **THEN** it SHALL calculate a pixel diff score against the downsampled source thumbnail
- **AND** the system SHALL iterate modifications to `scene_layout.json` until the diff score is `< 18.0`.

#### Scenario: Paths remain stable across platforms
- **WHEN** the pipeline runs on macOS, Linux, or Windows
- **THEN** the generated artifact locations SHALL be consistent and workspace-relative
- **AND** path separators visible to the user SHALL match the current platform’s conventions
